Everyone and everything that can act in the cooperative. Natural persons (members), legal entities (the LCA itself), and AI agents (Nou, Dianoia). Soft-deleted via archived_at — records are never hard-deleted.

agreements L1 LIVE REA · Contract — rows
Documents that create obligations: bylaws, membership agreements, grant letters, meeting minutes. Each agreement has a version and an effective date. When a new version supersedes an old one, superseded_by links them. Bylaws v2 is the only agreement currently loaded.

agreement_sections L1 LIVE REA · Clause — rows
Individual clauses within agreements, with stable addresses that survive version changes. The address format bylaws:v2:art-1.sec-3 lets RLS policies cite specific provisions by address. 187 sections of Bylaws v2 are loaded, from the Preamble through Article XVIII and Schedule A.

Time-bounded grants of cooperative roles to agents. A role assignment has a start date and optionally an end date. The RLS helper function has_role() queries this table to determine what any authenticated agent may see and do.

L2 · Audit
audit.events L2 LIVE Unified event stream
Append-only log of everything that happens in the CIS. Triggers on all L1 tables write to this stream automatically. Combines REA economic events (transfers, allocations) with operational audit trail (who changed what, when). Members can read their own events. Directors and admins can read all events.
iduuidPK
event_typetext NOT NULLe.g. INSERT · UPDATE · DELETE · economic_transfer

record_iduuidAffected row
actor_iduuidFK → agents.id (who did it)
old_datajsonbRow state before change
new_datajsonbRow state after change
metadatajsonbEvent context
created_attimestamptzEvent timestamp
Export Schema
export.* views LIVE Walkaway rehearsal exports
Read-only views in the export schema for generating CSV/JSON snapshots. Designed for the cooperative's walkaway rehearsal — a periodic test of whether members could reconstruct all essential records from the CIS. Views mirror L1 tables with human-readable field names.
Planned — Wave 2+
contributions W2 REA · Economic Event
The patronage engine — HOURS, REVENUE, CASH, OUTPUT, PRESENCE, PROPERTY primitives credited to member capital accounts. Six primitives, closed at the Bylaws level; adding a seventh requires amendment.
capital_accounts W2 REA · Economic Resource
Per-member capital balances, tracking allocated patronage, tax distributions, and redemption schedule. Governed by Subchapter K / IRC 704(b).
hub_bookings · spaces · memberships W3 Hub Operations
Co-working space management for 1515 Walnut — desk assignments, office leases, booking calendar, and Class Two/Three membership tracking.
