CIS Reference · Roles

Who can see and do what — and how roles are granted
Overview Schema Policies Roles Bylaws Migrations

The CIS defines 15 roles in the role_assignments table. Roles are time-bounded grants — each has a start date, an optional expiry, and a grantor. The RLS helper has_role() checks active assignments at query time.

Assignment mechanism: Role grants are written to role_assignments by directors or admins. Authentication is via Supabase Auth (W1 — not yet provisioned). Until W1, the CIS holds role data but cannot enforce it against real login sessions. Bootstrap roles assigned to AI agents are active.

Cooperative Core Roles

admin
System administrator. Full database access. Typically an AI agent or technical steward. Not a governance role — does not vote.
System role — not in Bylaws
director
Board member. Reads and writes all CIS data, creates agreements, grants roles. Elected by Class One members at annual meeting.
bylaws:v2:art-3 — Directors
officer
President, Vice President, Treasurer, or Secretary. Elevated write access for their functional area. Appointed by the board.
bylaws:v2:art-4 — Officers
patron_member
Active member with logged contributions. Eligible for patronage allocation at year-end. Reads the full resource table.
bylaws:v2:art-5.sec-1 — Patronage
member
Class One member in good standing. Reads member directory and agreements. One vote. $100 share.
bylaws:v2:art-1.sec-1 — Membership classes
applicant
Prospective member under board review. Reads bylaws and membership agreement only. No other access.
bylaws:v2:art-1.sec-2 — Admission
suspended_member
Member under suspension by board action. Read access reduced to bylaws only. Status reviewed at next board meeting.
bylaws:v2:art-1.sec-10 — Suspension
former_member
Withdrew or was terminated. Capital account on board-determined payout schedule (up to 5 years). Bylaws read access retained.
bylaws:v2:art-1.sec-11 — Termination
investor_member
Class Four investor. No voting unless co-held with Class One. No patronage. No workspace access. Terms pending Pote Law Firm.
bylaws:v2 — Class Four (pending)
community_supporter
Capital contribution without equity or membership rights. Governed by ULCAA § 7-58-104. No votes, no patronage.

Hub Operations Roles

hub_member
Class Two co-working member. $250/mo hot desk or $500+/mo dedicated. No voting. Hub MA drafting deferred.
bylaws:v2 — Class Two

Class Three participant. $25–$100/mo tiered (Threshold · Footing · Standing). No voting. Hub MA drafting deferred.
bylaws:v2 — Class Three

Staff managing the 1515 Walnut space. Operational write access to hub bookings and space tables (W3+).
Operational role — not a membership class
