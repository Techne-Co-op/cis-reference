The CIS is built incrementally. Each migration adds a defined layer. Three migrations constitute Wave 0 — the substrate. The wave model gives the cooperative a way to reason about what's live, what's planned, and what each increment unlocks.

Planned Waves

Phase 1 — Agent & Agreement PLANNED
Supabase Auth provisioned on this project. Member onboarding flow, directory, and profile. Stock ledger and signature tracking. Nou gains read-only directory and agreement lookup. CROPS targets: C·01 O·02 P·01 S·01.
New tables: memberships · stock_ledger · signatures
Auth: auth.uid() → agents.id via agent_id() helper
Bylaw basis: §1.1–1.4 · §1.13 · §1.7
Phase 2 — Capital Account PLANNED
Per-member capital account balance with full event history. Tax records for K-1 preparation. Multi-sig treasury structure. Nou gains balance queries and history summarization. CROPS targets: C·02 O·03 P·02 S·02.
New tables: capital_accounts · capital_events · tax_records
Patronage formula (proposed): 40% labor · 30% revenue · 20% cash · 10% community
Bylaw basis: §V · §VI · Subchapter K / IRC 704(b)
Phase 3 — Governance PLANNED
Meetings, motions, votes, minutes, and notices. Quorum calculation, vote threshold enforcement, consent workflows. On-chain attestation extended for selected events. Nou gains meeting context and motion drafting. CROPS targets: C·02 O·03 P·03 S·03.
New tables: meetings · attendance · proxies · motions · votes · consents · minutes · notices · committees
Bylaw basis: §II · §III · §X · §XII
Phase 4 — Lifecycle PLANNED
State transitions, redemption schedules, termination processes. Guided withdrawal, suspension tracking, admission flow refined. Nou gains lifecycle guidance and clock monitoring. CROPS targets: C·03 O·03 P·03 S·03.
Workflows: member admission · withdrawal · suspension · termination · redemption (up to 5 years)
Bylaw basis: §1.4 · §1.9 · §1.10 · §1.11
Phase 5 — Full Interface PLANNED
Derived views, analytics, benefit report generation. Complete member portal and public surface. Walkaway test runnable — cooperative can reconstruct all state from exported records. Nou full augmentation surface and audit support. CROPS targets: C·03 O·04 P·03 S·04.
New tables: benefit_reports · policies · conflict_disclosures
Walkaway test: all tables have documented export path; export schema in place from W0
Phase 6 — Practice PLANNED
No new schema migrations. Cultural and operational work to migrate CROPS properties from architecture into membership practice. Security hygiene becomes habitual. Openness becomes care for commons. Privacy understood as care for members. CROPS targets: C·04 O·04 P·04 S·04.
This phase lives in membership culture, not the database

Migration Repo

Migrations live in the CIS repository at supabase/migrations/. Each migration is a SQL file with a timestamp prefix and descriptive name. Applied via supabase db push or the Supabase dashboard migration runner.

The repo is private. When the intranet is rebuilt, migration history will be surfaced here automatically via the live-query layer on this page.
