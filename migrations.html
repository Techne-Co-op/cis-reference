The CIS is built incrementally. Each migration adds a defined layer. Three migrations constitute Wave 0 — the substrate. The wave model gives the cooperative a way to reason about what's live, what's planned, and what each increment unlocks.

Planned Waves

W1 — Authentication PLANNED
Supabase Auth provisioned on the CIS project. Map auth.uid() to agents.id via the agent_id() helper. Enable member login. Provision role_assignments for real members. RLS policies begin enforcing against live sessions.
Unlocks: Members can see their own directory entry, agreements, and audit trail
Requires: Auth provider config · member email list · invitation flow
W2 — Patronage Engine PLANNED
Capital accounts, contribution primitives (HOURS · REVENUE · CASH · OUTPUT · PRESENCE · PROPERTY), and year-end allocation. The patronage formula maps to K-1 distributions. A validated engine with 184 tests exists; needs migration to this schema.
New tables: contributions · capital_accounts · allocations
Six primitives: closed at Bylaws level (art-5)
Formula: 40% labor · 30% revenue · 20% cash · 10% community
W3 — Treasury PLANNED
Financial statement integration. Mercury bank account, Stripe payouts, and Xero entries linked to CIS resources. Monthly statement generation. The June 2026 statement currently exists as a standalone HTML file; W3 bakes this into the CIS.
Integrations: Mercury API · Stripe API · Xero OAuth
New tables: ledger_entries · bank_accounts · statement_periods
W4 — People PLANNED
Full member onboarding flow. Invitation, application, board review, admission, share purchase — all tracked in the CIS. Membership Agreement v2.3 loaded as an agreement record. Class One admission tied to role_assignments.
New tables: applications · invitations · share_purchases
Requires: W1 (auth) complete
W5 — Hub Operations PLANNED
1515 Walnut co-working space management. Desk assignments, office leases, Class Two/Three membership billing, booking calendar. Revenue tracked into W3 treasury layer.
New tables: spaces · bookings · hub_memberships · space_revenue
Target MRR: $10-11K/mo by June 2027
W6 — Coordination PLANNED
Decision threads, deliberation notes, and the cooperative's governance coordination surface. Replaces the co-op.us Workshop system. The intranet rebuild will surface this as the member-facing coordination layer.
New tables: decision_threads · deliberation_notes · agenda_items
Replaces: hvbdpgkdcdskhpbdeeim (co-op.us Workshop — being retired)

Migration Repo

Migrations live in the CIS repository at supabase/migrations/. Each migration is a SQL file with a timestamp prefix and descriptive name. Applied via supabase db push or the Supabase dashboard migration runner.

The repo is private. When the intranet is rebuilt, migration history will be surfaced here automatically via the live-query layer on this page.
